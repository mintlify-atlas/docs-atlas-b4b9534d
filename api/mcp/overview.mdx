## Introduction

Entendi provides a comprehensive MCP server that enables AI assistants to track concept mastery, issue comprehension probes, and provide adaptive tutoring. The server implements the [Model Context Protocol](https://modelcontextprotocol.io) for seamless integration with Claude Code and other MCP-compatible clients.

## Architecture

```
Claude Code ──► MCP Server (stdio) ──► HTTP API (Hono) ──► Neon PostgreSQL
```

The MCP server acts as a bridge between your AI assistant and Entendi's backend:
- **Transport**: stdio (standard input/output)
- **Authentication**: API key-based (stored in `~/.entendi/config.json`)
- **State Management**: Delegated to the production API
- **Resilience**: Built-in retry logic, circuit breaker, and response caching

## Authentication Flow

<Steps>
  <Step title="Initial Login">
    Call `entendi_login` without parameters to generate a device code and open browser
  </Step>
  <Step title="Browser Confirmation">
    Sign in to Entendi and confirm device link
  </Step>
  <Step title="Retrieve API Key">
    Call `entendi_login` again with the device code to poll for confirmation and save API key
  </Step>
  <Step title="Auto-Configuration">
    API key is saved to `~/.entendi/config.json` and loaded automatically on server restart
  </Step>
</Steps>

## Error Handling

All MCP tools return user-friendly error messages wrapped by `wrapToolError()`. Common scenarios:

| Error Type | User Message |
|------------|-------------|
| Circuit breaker open | "Entendi is temporarily unavailable. Your work continues normally — concept tracking will resume automatically." |
| Network failures | "Can't reach the Entendi API right now. This doesn't affect your work." |
| Expired session (401/403) | "Your Entendi session has expired. Run `entendi_login` to re-authenticate." |
| Rate limit (429) | "Rate limit reached. Try again later." |
| Server errors (5xx) | "Entendi server error. Your work continues normally." |
| Expired probe token | "This probe has expired. A new one will be issued next time." |

## Resilience Features

### Circuit Breaker

Protects against cascading failures:
- **Threshold**: 5 consecutive failures
- **Cooldown**: 30 seconds before half-open probe
- **States**: `closed` → `open` → `half-open` → `closed`

### Retry Logic

Automatic exponential backoff with jitter:
- **Max retries**: 3 attempts
- **Base delay**: 1000ms
- **Jitter**: ±25%
- **Retryable codes**: 429 (rate limit), 5xx (server errors)

### Response Caching

Read-only endpoints cached for 60 seconds:
- `entendi_get_status`
- `entendi_get_zpd_frontier`

Cache invalidated automatically after write operations.
