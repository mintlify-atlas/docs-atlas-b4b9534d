## Overview

The `entendi_record_evaluation` tool updates mastery state after a comprehension probe or tutor phase. It uses:

- **Bayesian inference** with the Graded Response Model (GRM) to update mastery (`mu`, `sigma`)
- **Time decay** (FSRS) to adjust for forgetting since last assessment
- **Counterfactual tracking** to separate tutored vs. untutored mastery
- **Tutor triggering** to determine if Socratic tutoring should be offered

## Bayesian Update Process

### 1. Time Decay (FSRS)

If the concept was previously assessed, apply forgetting curve:

```typescript
function applyTimeDecay(mu: number, sigma: number, daysSince: number, stability: number): { mu: number, sigma: number } {
  const retrievability = Math.exp(-daysSince / stability);
  const decayedMu = mu * retrievability;
  const decayedSigma = Math.sqrt(sigma ** 2 + (1 - retrievability) * 0.5);
  return { mu: decayedMu, sigma: decayedSigma };
}
```

### 2. GRM Likelihood

Compute probability of observing score `k` given mastery `θ`:

```typescript
function grmProbability(score: 0 | 1 | 2 | 3, theta: number, itemParams: GRMItemParams): number {
  const { discrimination: a, thresholds: b } = itemParams;
  
  // P(Y ≥ k | θ) = 1 / (1 + exp(-a(θ - b[k-1])))
  const pAbove = (k: number) => k === 0 ? 1 : 1 / (1 + Math.exp(-a * (theta - b[k - 1])));
  
  return pAbove(score) - pAbove(score + 1);
}
```

### 3. Posterior Update

Combine prior (time-decayed mastery) with likelihood (GRM) using Laplace approximation:

```typescript
function bayesianUpdate(prior: { mu: number, sigma: number }, score: number, confidence: number, itemParams: GRMItemParams) {
  // Log-posterior = log-prior + confidence * log-likelihood
  const logPosterior = (theta: number) => {
    const logPrior = -0.5 * ((theta - prior.mu) / prior.sigma) ** 2;
    const logLikelihood = Math.log(grmProbability(score, theta, itemParams));
    return logPrior + confidence * logLikelihood;
  };
  
  // Find MAP estimate and compute Hessian for σ
  const map = findMaximum(logPosterior);
  const hessian = numericalHessian(logPosterior, map);
  const posteriorSigma = 1 / Math.sqrt(-hessian);
  
  return { mu: map, sigma: posteriorSigma };
}
```

### 4. Counterfactual Tracking

If `eventType` is `tutor_phase1` or `tutor_phase4`, maintain separate untutored estimates:

```typescript
if (eventType === 'tutor_phase1') {
  // Record pre-tutoring mastery as untutored baseline
  ucs.muUntutored = ucs.mastery.mu;
  ucs.sigmaUntutored = ucs.mastery.sigma;
} else if (eventType === 'tutor_phase4') {
  // Don't update untutored estimates (shows tutoring effect)
  ucs.tutoredAssessmentCount++;
} else {
  // Regular probe: update both tutored and untutored
  ucs.muUntutored = posterior.mu;
  ucs.sigmaUntutored = posterior.sigma;
  ucs.untutoredAssessmentCount++;
}
```

## Tutor Triggering

For `eventType: "probe"`, determine if tutoring should be offered:

| Tutor Mode | Trigger Condition |
|------------|-------------------|
| `manual` | Never auto-trigger |
| `reactive` | `score ≤ tutorTriggerThreshold` |
| `proactive` | Always offer tutoring |

Default: `reactive` with `tutorTriggerThreshold: 1` (trigger on scores 0-1).

## Side Effects

1. **Mastery Update**: Updates `mu`, `sigma` for the concept's user state
2. **Memory Update**: Updates FSRS `stability` and `difficulty` based on performance
3. **Assessment History**: Appends `AssessmentEvent` to `history` array
4. **Pending Action**: Clears `awaiting_probe_response` pending action
5. **Tutor Offer**: Writes `tutor_offered` pending action if `shouldOfferTutor` is true
6. **Cache Invalidation**: Invalidates `/api/mcp/status` and `/api/mcp/zpd-frontier` caches

## Integration Pattern

```typescript
// After user responds to probe
const evaluation = await llm.evaluateResponse(userResponse, probeCriteria);

const result = await mcpClient.callTool('entendi_record_evaluation', {
  conceptId: pendingProbe.conceptId,
  score: evaluation.score,
  confidence: evaluation.confidence,
  reasoning: evaluation.reasoning,
  eventType: 'probe',
  responseText: userResponse,
});

if (result.shouldOfferTutor) {
  await offerTutorSession(pendingProbe.conceptId);
}
```
