## Prerequisites

Before starting development, ensure you have:

- **Node.js 22+** (check with `node --version`)
- **Git** (for version control)
- **PostgreSQL** access (Neon serverless recommended)
- **Claude Code CLI** installed (for plugin testing)
- **Optional**: Docker + VS Code/Cursor with Dev Containers extension

## Initial Setup

### Automated Setup (Recommended)

```bash
git clone https://github.com/LemonThyme/entendi.git
cd entendi
./setup.sh
```

The `setup.sh` script:
1. Validates Node.js version (requires 22+)
2. Installs npm dependencies
3. Runs the build pipeline
4. Installs the plugin to `~/.claude/plugins/entendi`
5. Creates `~/.entendi/` directory for local state

### Manual Setup

For manual setup or CI environments:

```bash
npm install
cp .env.example .env   # Edit with your DATABASE_URL and secrets
npm run build
```

## Environment Configuration

### Required Variables

Create a `.env` file in the project root:

```bash
# Database (required)
DATABASE_URL="postgresql://user:pass@host/dbname?sslmode=require"

# Auth (required)
BETTER_AUTH_SECRET="$(openssl rand -base64 32)"
BETTER_AUTH_URL="http://localhost:3456"

# Used by hooks/MCP (should match BETTER_AUTH_URL)
ENTENDI_API_URL="http://localhost:3456"
```

### Optional Variables

```bash
# OAuth Providers
GITHUB_CLIENT_ID="your_github_client_id"
GITHUB_CLIENT_SECRET="your_github_client_secret"
GOOGLE_CLIENT_ID="your_google_client_id"
GOOGLE_CLIENT_SECRET="your_google_client_secret"

# Payments (Stripe)
```

## Development Workflow

### 1. Start the API Server

```bash
npm run api:dev
```

This starts a local Hono server on `http://localhost:3456` with hot-reload.

**What it does**:
- Loads `.env` variables
- Connects to PostgreSQL via Drizzle ORM
- Serves API routes (`/api/*`)
- Serves dashboard (`/`, `/dashboard`)
- Logs all requests to console

### 2. Make Code Changes

Edit source files in:
- `src/api/` — HTTP API and routes
- `src/core/` — Business logic (GRM, probes, tutors)
- `src/mcp/` — MCP server
- `src/hooks/` — Claude Code hooks
- `src/dashboard/` — Dashboard UI
- `plugin/` — Plugin metadata and static files

### 3. Rebuild and Reinstall Plugin

After making changes to hooks, MCP, or plugin:

```bash
npm run plugin:reinstall
```

This command:
1. Runs `npm run build` to rebuild all components
2. Clears Claude Code's plugin cache (`~/.claude/plugins/cache/entendi`)
3. Installs the plugin from `dist/plugin/` to `~/.claude/plugins/entendi`

**Note**: The build stamps the plugin version with the current git hash, so Claude Code will always pick up your latest changes.

### 4. Test in Claude Code

Restart Claude Code:

```bash
claude
```

The plugin activates automatically. Test by:

```
You: "entendi status"
Claude: [shows current session state]
```

### 5. Check Debug Logs

All hooks, MCP tools, and API calls log to:

```bash
tail -f ~/.entendi/debug.log
```

**Log Format**:
```
[2026-03-02T14:23:45.123Z] [UserPromptSubmit] User prompt: "deploy to Cloudflare Workers"
[2026-03-02T14:23:45.234Z] [MCP:entendi_observe] Concepts extracted: ["cloudflare-workers", "v8-isolates"]
[2026-03-02T14:23:46.345Z] [API:POST /api/mcp/observe] 200 OK (234ms)
```

## Build System Deep Dive

### Build Pipeline

The build is orchestrated by `esbuild.config.ts` and runs four parallel builds:

```bash
npm run build
```

Equivalent to:
```bash
npx tsc && node --import tsx esbuild.config.ts
```

#### 1. TypeScript Compilation

```bash
npx tsc
```

Compiles all `.ts` files to `.js` using `tsconfig.json`. Output goes to `dist/` (but is overwritten by esbuild for bundled components).

#### 2. Hook Bundles

Bundles hooks from `src/hooks/*.ts` to standalone JS files:

```typescript
await esbuild.build({
  entryPoints: hookFiles,
  bundle: true,
  platform: 'node',
  target: 'node22',
  outdir: 'dist/hooks',
  format: 'esm',
  banner: { js: '#!/usr/bin/env node' },
  external: ['@anthropic-ai/sdk'],
});
```

**Output**: `dist/hooks/user-prompt-submit.js`, `dist/hooks/notification.js`, etc.

#### 3. MCP Server Bundle

Bundles the MCP server to a single executable:

```typescript
await esbuild.build({
  entryPoints: [join('src', 'mcp', 'server.ts')],
  bundle: true,
  platform: 'node',
  target: 'node22',
  outdir: 'dist/mcp',
  format: 'esm',
  banner: { js: '#!/usr/bin/env node' },
  external: ['@anthropic-ai/sdk'],
});
```

**Output**: `dist/mcp/server.js` (chmod +x)

#### 4. Plugin Assembly

```typescript
// Clean previous build
rmSync(pluginDir, { recursive: true, force: true });

// Copy static plugin files
copyDir('plugin', pluginDir);

// Stamp version with git hash
const gitHash = execFileSync('git', ['rev-parse', '--short', 'HEAD']).trim();
pluginMeta.version = `${pluginMeta.version}+${gitHash}`;

// Copy built JS bundles
copyFileSync('dist/hooks/user-prompt-submit.js', join(pluginDir, 'hooks', 'user-prompt-submit.js'));
copyFileSync('dist/mcp/server.js', join(pluginDir, 'mcp', 'server.js'));
```

**Output**: `dist/plugin/` (ready for `claude plugin install`)

#### 5. Dashboard Assets

Builds dashboard JS/CSS with content hashing:

```typescript
const dashboardBuild = await esbuild.build({
  entryPoints: [
    'src/dashboard/dashboard.js',
    'src/dashboard/dashboard.css',
    'src/dashboard/link.js',
  ],
  bundle: false,
  minify: true,
  outdir: 'public/assets',
  entryNames: '[name]-[hash]',
  metafile: true,
});
```

**Output**: `public/assets/dashboard-[hash].js`, `public/assets/dashboard-[hash].css`

**Manifest Generation**: A `manifest.json` maps original names to hashed filenames:

```json
{
  "dashboard.js": "/assets/dashboard-Q7X2K9ZY.js",
  "dashboard.css": "/assets/dashboard-A3B8F1D2.css",
  "charts.js": "/assets/charts-Z9Y4L6M1.js"
}
```

This manifest is imported by `src/dashboard/manifest.ts` for Workers to serve the correct asset URLs.

### Build Scripts

| Script | Purpose |
|--------|--------|
| `npm run build` | Full build (tsc + esbuild) |
| `npm run build:hooks` | Rebuild hooks only |
| `npm run plugin:reinstall` | Build + clear cache + reinstall |

## Testing

### Unit Tests

```bash
npm test              # Run all tests once
npm run test:watch    # Watch mode
```

**Framework**: Vitest

**Test Files**: `src/**/*.test.ts`

Example test structure:

```typescript
import { describe, expect, it } from 'vitest';
import { retrievability } from './probabilistic-model';

describe('retrievability', () => {
  it('returns 1.0 at t=0', () => {
    expect(retrievability(0, 10)).toBe(1.0);
  });

  it('returns 0.9 at t=S', () => {
    expect(retrievability(10, 10)).toBeCloseTo(0.9, 2);
  });
});
```

This script validates:

## Contributing Workflow

### Commit Convention

Entendi uses [Conventional Commits](https://www.conventionalcommits.org/) for automated versioning.

**Required Format**: `<type>(<scope>): <description>`

| Type | When to Use | Version Bump |
|------|------------|-------------|
| `feat:` | New feature or capability | minor (0.1.0 → 0.2.0) |
| `fix:` | Bug fix | patch (0.1.0 → 0.1.1) |
| `perf:` | Performance improvement | patch |
| `docs:` | Documentation only | none |
| `test:` | Adding/updating tests | none |
| `chore:` | Maintenance, deps, config | none |
| `refactor:` | Code refactoring | none |
| `feat!:` | Breaking change | minor (capped while pre-1.0) |

**Examples**:
```bash
git commit -m "feat(probe): add counterfactual question type"
git commit -m "fix(auth): handle expired device codes"
git commit -m "docs: update development guide"
```

**Rules**:
- Prefix is **mandatory**
- Use imperative mood: "add", "fix", "update" (not "added", "fixes")
- Keep subject under 72 characters
- Scope is optional but encouraged: `feat(api)`, `fix(dashboard)`

### Pull Request Workflow

1. **Create Feature Branch**:
   ```bash
   git checkout -b feat/my-feature
   ```

2. **Make Changes**:
   - Write code
   - Add tests
   - Update docs if needed

3. **Lint and Test**:
   ```bash
   npm run lint
   npm test
   ```

4. **Commit with Convention**:
   ```bash
   git add .
   git commit -m "feat(core): add new probe type"
   ```

5. **Push and Open PR**:
   ```bash
   git push origin feat/my-feature
   ```

6. **Wait for CI**:
   - Linting (Biome)
   - Tests (Vitest)
   - Build validation

### Code Quality

**Linter**: Biome (replaces ESLint + Prettier)

**Type Checking**:

## Common Development Tasks

### Add a New MCP Tool

1. **Define Tool Schema** in `src/mcp/server.ts`:
   ```typescript
   const tools = [
     // ... existing tools
     {
       name: 'entendi_my_tool',
       description: 'Does something useful',
       inputSchema: {
         type: 'object',
         properties: {
           param: { type: 'string' },
         },
         required: ['param'],
       },
     },
   ];
   ```

2. **Implement Handler**:
   ```typescript
   async function handleToolCall(name: string, args: any) {
     if (name === 'entendi_my_tool') {
       const result = await fetch(`${apiUrl}/api/mcp/my-tool`, {
         method: 'POST',
         headers: { 'Authorization': `Bearer ${apiKey}` },
         body: JSON.stringify(args),
       });
       return result.json();
     }
   }
   ```

3. **Add API Route** in `src/api/routes/mcp.ts`:
   ```typescript
   app.post('/api/mcp/my-tool', async (c) => {
     const { param } = await c.req.json();
     // Business logic here
     return c.json({ success: true });
   });
   ```

4. **Rebuild and Test**:
   ```bash
   npm run plugin:reinstall
   claude
   ```

### Add a New Hook

1. **Create Hook File** in `src/hooks/my-hook.ts`:
   ```typescript
   #!/usr/bin/env node
   import { readFileSync } from 'fs';
   
   const input = readFileSync(0, 'utf-8');
   const context = JSON.parse(input);
   
   // Hook logic here
   
   console.log(JSON.stringify({ status: 'success' }));
   ```

2. **Add to Build Config** in `esbuild.config.ts`:
   ```typescript
   const hookEntrypoints = [
     'user-prompt-submit.js',
     'notification.js',
     'my-hook.js',  // Add here
   ];
   ```

3. **Create Bash Wrapper** in `plugin/hooks/my-hook`:
   ```bash
   #!/bin/bash
   "$PLUGIN_DIR/hooks/my-hook.js" "$@"
   ```

4. **Register in plugin.json**:
   ```json
   {
     "hooks": {
       "MyHook": "hooks/my-hook"
     }
   }
   ```

### Update Dashboard UI

1. **Edit** `src/dashboard/dashboard.js` or `dashboard.css`
2. **Rebuild Assets**: `npm run build`
3. **Refresh Browser**: Hard reload (`Cmd+Shift+R`)

**Note**: Dashboard assets are content-hashed, so you may need to clear browser cache.

## Troubleshooting

### Plugin Not Loading

**Symptom**: Claude Code doesn't recognize Entendi commands

**Solution**:
1. Check plugin is installed: `ls ~/.claude/plugins/entendi`
2. Verify `plugin.json` is valid: `cat ~/.claude/plugins/entendi/.claude-plugin/plugin.json | jq`
3. Clear cache: `rm -rf ~/.claude/plugins/cache/entendi`
4. Reinstall: `npm run plugin:reinstall`
5. Restart Claude Code

### Database Connection Errors

**Symptom**: `Error: connect ECONNREFUSED` or SSL errors

**Solution**:
1. Verify `DATABASE_URL` in `.env`
2. Test connection: `psql "$DATABASE_URL"`
3. Check Neon project is active (serverless may suspend)
4. Add `?sslmode=require` to connection string

### Build Failures

**Symptom**: `npm run build` fails with TypeScript errors

**Solution**:
1. Clear `dist/`: `rm -rf dist/`
2. Clear node_modules: `rm -rf node_modules && npm install`
3. Check Node version: `node --version` (must be 22+)
4. Fix TypeScript errors: `npx tsc --noEmit`
