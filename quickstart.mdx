This guide walks you through installing Entendi, linking your account, and experiencing your first comprehension probe.

## Install Entendi

<Steps>
  <Step title="Install Claude Code CLI">
    If you don't have it already:
    
    ```bash
    npm install -g @anthropic-ai/claude-code
    ```
  </Step>

  <Step title="Add Entendi from GitHub">
    ```bash
    claude plugin marketplace add https://github.com/LemonThyme/entendi
    claude plugin install entendi
    ```
  </Step>

  <Step title="Start Claude Code">
    ```bash
    claude
    ```
    
    You should see Claude Code start up. The Entendi plugin is now active, but not yet authenticated.
  </Step>
</Steps>

## Link Your Account

Inside the Claude Code session, authenticate with Entendi:

<Steps>
  <Step title="Request login">
    ```
    You: entendi login
    ```
  </Step>

  <Step title="Confirm in browser">
    Claude will call the `entendi_login` MCP tool, which:
    
    1. Creates a device code (8-character alphanumeric, like `A3K7M9Q2`)
    2. Opens your browser to `https://api.entendi.dev/link?code=A3K7M9Q2`
    3. Shows you the code and expiry time
    
    **In the browser:**
    - Sign in (or create an account if this is your first time)
    - Click **Confirm Link**
    - You'll see "Device linked successfully"
  </Step>

  <Step title="Tell Claude you're done">
    ```
    You: I confirmed it
    ```
    
    Claude will poll the device code status and retrieve your API key. The key is automatically saved to `~/.entendi/config.json`.
  </Step>

  <Step title="Restart Claude Code">
    Exit and restart:
    
    ```bash
    # Press Ctrl+D or type "exit"
    claude
    ```
    
    The plugin will now load with full authentication.
  </Step>
</Steps>

## Your First Probe

Entendi watches your coding session and probes your understanding when you work with new or rusty concepts.

<Steps>
  <Step title="Trigger concept detection">
    Ask Claude to help with something technical:
    
    ```
    You: Set up a Docker container for this project
    ```
    
    Claude completes the request (creates a Dockerfile, docker-compose.yml, etc.).
  </Step>

  <Step title="Get probed">
    After finishing, Claude asks a comprehension question:
    
    ```
    Claude: [completes the Docker setup]
    
    🧠 Entendi — Docker Containers
    ───────────────────────────────────────
    What's the difference between an image and a container,
    and why do we use docker-compose here instead of a single
    Dockerfile?
    ───────────────────────────────────────
    ```
  </Step>

  <Step title="Respond naturally">
    ```
    You: An image is like a template, and a container is a running
    instance of that image. We use docker-compose because it lets
    us define multiple services and their relationships in one file.
    ```
    
    Claude evaluates your response on a 0-3 rubric:
    - **0** — No understanding
    - **1** — Surface-level (buzzwords, no depth)
    - **2** — Functional (can use it correctly)
    - **3** — Deep/transferable (can explain trade-offs and alternatives)
  </Step>

  <Step title="Get feedback">
    If you score 0-1, Claude offers a tutor session:
    
    ```
    Claude: I see some gaps. Want me to walk you through it?
    
    You: Sure
    
    Claude: [starts 4-phase Socratic tutor]
    Phase 1: What do you already know about Docker?
    ...
    ```
    
    If you score 2-3, Claude updates your mastery and moves on.
  </Step>
</Steps>

## What Just Happened?

Behind the scenes, Entendi:

1. **Detected concepts** — The `UserPromptSubmit` hook extracted "docker" and "docker-compose" from your prompt
2. **Checked mastery** — Called `entendi_observe` to query your Bayesian knowledge state for those concepts
3. **Decided to probe** — Selected "docker" as the highest-value probe based on uncertainty and information gain
4. **Issued a probe** — Created a signed `probeToken` with depth, criteria, and expiry
5. **Evaluated your answer** — Scored your response and called `entendi_record_evaluation` to update your knowledge graph
6. **Offered tutor** — Since you scored low, it triggered `entendi_start_tutor` to help you learn

## Check Your Knowledge Profile

View your mastery levels at [entendi.dev](https://entendi.dev):

- **Mastered** — Concepts with ≥70% mastery
- **In Progress** — Concepts you've been assessed on but not mastered
- **ZPD Frontier** — Zone of Proximal Development: what you're ready to learn next

## CLI Commands

All Entendi interactions happen through natural conversation with Claude, but you can also use MCP tools directly:

```
You: Call entendi_get_status with conceptId "docker"
```

Claude will show:

```json
{
  "concept": {
    "mastery": 0.42,
    "sigma": 0.8,
    "confidenceLevel": "medium",
    "assessmentCount": 3,
    "lastAssessed": "2026-03-02T14:23:11.000Z",
    "urgency": 0.65
  }
}
```

## Customizing Behavior

Entendi adapts its intrusiveness based on your profile:

- **Unknown** (0 assessments) — Direct probes for novel concepts
- **Beginner** (avg mastery \<40%) — Direct probes for novel, woven for adjacent
- **Intermediate** (40-75%) — Woven probes for novel and adjacent
- **Advanced** (\>75%) — Woven probes for novel only, skips adjacent

**Woven** means the probe is embedded naturally in Claude's response. **Direct** means it's explicitly marked with the 🧠 Entendi banner.

## Troubleshooting

### "No API key found" after login

Restart Claude Code:

```bash
# Exit current session (Ctrl+D)
claude
```

The MCP server loads `~/.entendi/config.json` at startup.

### Probes not appearing

Check enforcement level:

```
You: Call entendi_health_check
```

If `enforcement` is `off`, probes are disabled. Contact your org admin to change it.

### Login link doesn't open

Manually visit:

```
https://api.entendi.dev/link?code=<your-code>
```

Replace `<your-code>` with the 8-character code shown in Claude's response.
